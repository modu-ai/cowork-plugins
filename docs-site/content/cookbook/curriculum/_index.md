# 커리큘럼 개요

> Claude Cowork를 실무에 완전히 통합하여 업무 효율성을 극대화하는 종합 과정입니다.

## 커리큘럼 개요

이 커리큘럼은 Claude Cowork 초보자가 실무 전문가로 성장하기 위한 8주 단계별 학습 경로입니다. 각 주차는 이전 주차 내용을 기반으로 점진적으로 심화됩니다.

### 주차별 학습 흐름

| 주차 | 주제 | 핵심 목표 | 결과물 |
|------|------|----------|--------|
| **1-2주차** | Cowork 기초 & 단일 스킬 | [랩 1](lab-1/) - 재무 대리 AI 만들기 | `finance-assistant` 플러그인 + 3개 스킬 |
| **3-4주차** | 스킬 체이닝 & 자동화 | [랩 2](lab-2/) - 품질·SCM AI 만들기 | `quality-ops` + `scm-watcher` 플러그인 |
| **5-6주차** | AI 사원 설계 & 도메인 트랙 | [AI 사원 설계](../ai-employee-design/) | 본인 업무에 맞는 AI 사원 설계 능력 |
| **7-8주차** | 최종 프로젝트 | [최종 프로젝트](final-project/) | 3-스킬 체인으로 자동화한 실제 업무 |

### 단계별 학습 목표

#### **Phase 1: 기초 다지기 (1-2주차)**
- **Cowork 기본 활용법** 설치부터 MCP 연결까지
- **단일 스킬 개발** 플러그인 스캐폴드부터 Schedule 등록까지
- **실무 적용** 재무팀 반복 업무 자동화 사례 학습

#### **Phase 2: 체이닝 마스터리 (3-4주차)**
- **다중 스킬 설계** 도메인 → 포맷 → 품질 3단계 체인
- **실시간 모니터링** 이상 탐지 및 긴급 알림 시스템
- **데이터 연동** Excel, CSV, 외부 API 통합 방법

#### **Phase 3: 전문화 (5-6주차)**
- **도메인 전문가** 재무, 품질, 마케팅 등 전문 분야 AI 설계
- **복잡한 워크플로우** 조건부 실행, 분기 로직 설계
- **최적화 기법** 토큰 효율성, 병렬 처리, 오류 처리

#### **Phase 4: 실전 적용 (7-8주차)**
- **종합 프로젝트** 본인 업무 1건 완전 자동화
- **품질 관리** 루브릭 기반 자가 평가
- **지식 공유** 결과물 문서화 및 동료 리뷰

## 학습 로드맵

### 주 1-2: Cowork 입문 (랩 1)
- **학습 내용**: 플러그인 생성, 스킬 개발, Schedule 등록
- **실습 과제**: 재무 대리 AI 만들기 (3개 스킬)
- **주요 기술**: WebFetch, Gmail MCP, Slack 알림
- **결과물**: 매일 자동 실행되는 재무 브리핑 시스템

### 주 3-4: 체이닝 심화 (랩 2)
- **학습 내용**: 복수 스킬 설계, 데이터 분석, 실시간 모니터링
- **실습 과제**: 품질·SCM AI 만들기 (4개 스킬)
- **주요 기술**: Excel 데이터 처리, 이상 탐지, 벤더 리스크 분석
- **결과물**: 공장 품질 대시보드 + 벤더 모니터링 시스템

### 주 5-6: AI 사원 설계
- **학습 내용**: 도메인별 AI 설계 패턴, 전문화된 스킬 개발
- **실습 과제**: 본인 업무 분석 및 AI 설계
- **주요 기술**: 도메인 모델링, 워크플로우 최적화
- **결과물**: 개인화된 AI 사원 설계서

### 주 7-8: 최종 프로젝트
- **학습 내용**: 종합 적용, 품질 평가, 지식 공유
- **실습 과제**: 실제 업무 1건 3-스킬 체인으로 자동화
- **주요 기술**: 체인 설계, 프롬프트 최적화, 결과 평가
- **결과물**: 자동화된 업무 시스템 + 평가 보고서

## 평가 기준

### 단계별 평가
- **랩 1**: 3개 스킬 모두 정상 실행 + Schedule 등록 완료
- **랩 2**: 4개 스킬 정상 실행 + 이상 탐지 로직 동작 확인
- **AI 사원 설계**: 도메인 분석서 완성 + 스킬 체인 설계서
- **최종 프로젝트**: 루브릭 12점 이상 + 동료 리뷰 완료

### 합격 기준
- 모든 Schedule이 자동으로 실행되어야 함
- MCP 연결이 안정적으로 유지되어야 함
- 생성된 산출물이 실제 사용 가능한 수준이어야 함
- 오류 발생 시 자체 복구 메커니즘이 포함되어야 함
