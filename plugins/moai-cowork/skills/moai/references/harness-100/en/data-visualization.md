# Data Visualization (harness-100-data-visualization)

> MoAI-Cowork V.0.1.3 Harness Reference

## Overview
The Data Visualization harness empowers analysts, managers, and decision-makers to transform raw data into compelling visual stories that drive action. Whether creating dashboards, selecting chart types, designing executive summaries, or implementing accessibility standards, this harness guides you through visual design principles and data storytelling techniques. Perfect for anyone communicating insights to stakeholders, from marketers analyzing campaign performance to executives reviewing quarterly metrics.

## Expert Roles
**1. Chart Type Strategist**
Selects optimal visualization formats for different data relationships. Responsibilities: analyze data type and distribution, choose appropriate chart types, design multi-dimensional visualizations, recommend interactive elements, evaluate clarity and impact.

**2. Dashboard Designer**
Architects comprehensive information displays for monitoring and analysis. Responsibilities: define KPI framework and metrics hierarchy, design dashboard layout and flow, establish information priority and drill-down paths, design responsive layouts, create dashboard templates.

**3. Data Storyteller**
Crafts narratives that guide audiences through insights and implications. Responsibilities: identify key insights and business implications, sequence visual elements for narrative flow, write compelling titles and annotations, design slide-by-slide story arcs, prepare presentation narratives.

**4. Accessibility & Color Specialist**
Ensures visualizations are inclusive and effective across audiences. Responsibilities: select accessible color palettes, design for colorblind users, optimize for various devices, ensure readability and contrast, test with diverse user groups.

**5. Executive Summary Architect**
Distills complex data into high-level insights for decision-makers. Responsibilities: identify critical metrics and trends, design one-page executive summaries, highlight key takeaways and recommendations, create snapshot visualizations, design comparative views.

## Workflow
### Phase 1: Context Gathering
- Define visualization objectives and key decisions to support
- Identify audience type, technical literacy, and information needs
- Gather data sources, data quality, and update frequency
- Establish performance metrics and success criteria
- Document design constraints and technology platforms

### Phase 2: Analysis & Production
- Analyze data structure, relationships, and outliers
- Select optimal chart types for each insight
- Design dashboard layouts and information hierarchies
- Create accessible color palettes and design systems
- Draft narrative annotations and insight interpretations

### Phase 3: Review & Delivery
- Test visualizations with target audience members
- Validate accuracy and completeness of data representation
- Optimize performance and load times
- Create documentation for updating and maintaining visualizations
- Prepare presentation guidelines and usage instructions

## Deliverables
- **Chart Type Recommendations:** Selected visualizations with rationale and alternatives
- **Dashboard Design:** Layout mockups, wireframes, and component specifications
- **Data Story Presentation:** Slide sequence with supporting visuals and narrative script
- **Accessibility Audit:** Color palette analysis, contrast checks, and recommendations
- **Executive Summary:** One-page visual distillation of key insights and metrics

## Extension Skills
- **Interactive Dashboard Development:** Build drill-down and filtering capabilities
- **Real-Time Data Monitoring:** Design dashboards for live streaming and alerts
- **Comparative Visualization:** Design side-by-side comparisons and performance benchmarks

## Error Handling
| Error | Cause | Solution |
|-------|-------|----------|
| Cluttered Dashboard | Too many metrics or poor information hierarchy | Remove non-critical metrics, reorganize by business priority, create multiple focused views |
| Misleading Visualization | Inappropriate chart type or axis manipulation | Verify data accuracy, select chart aligned with data, use honest scale representation |
| Low Adoption | Unclear insights or difficult navigation | Simplify design, add guided interpretation, provide user training |
| Accessibility Failures | Color-dependent design or poor contrast | Use accessible color palettes, add patterns/textures, test with accessibility tools |
