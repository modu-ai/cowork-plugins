# 강의 운영 매뉴얼 자동 생성

## 개요

PDF §8 직접 인용:

> "운영 매뉴얼 — 시간표·실습 운영·강사 동선·리스크"

강의 기획서 또는 강사 한 줄 컨셉을 입력받아 3일 21세션 완전한 운영 매뉴얼을 자동 생성합니다. D-30 운영 준비부터 D-7 사전 준비물 메일까지 캠프 운영 전 과정을 커버합니다.

**체이닝 출력**
운영 매뉴얼 생성 후 `moai-office:docx-generator`를 자동 체이닝하여 Word(.docx) 형식으로 저장합니다.

**합격 기준 (PDF §8 직접 인용)**
PDF §8 모든 섹션(8.1~8.6) 포함 + Plan B 시나리오 5건 이상.

## 워크플로우

### 호출 방법

```bash
# 전체 운영 매뉴얼 생성 (기본)
/course-curriculum-design --course "모두의커머스 3일 마스터 캠프"

# D-7 사전 준비물 메일 출력
/course-curriculum-design --output d7-mail

# 긴급 Plan B 추출 (실시간 리스크 대응)
/course-curriculum-design --emergency MAX_PATH

# 일수·세션 수 커스텀
/course-curriculum-design --course "1일 AI 기초" --days 1 --sessions 7
```

### 입력 슬롯

| 슬롯 | 필수 여부 | 기본값 | 설명 |
|------|-----------|--------|------|
| `--course` | 선택 | 없음 | 강의명 또는 한 줄 컨셉 |
| `--days` | 선택 | 3 | 강의 일수 (1~5) |
| `--sessions` | 선택 | 7 | 일별 세션 수 (기본 7 = S1~S7 × 50분) |
| `--output` | 선택 | full | full / d7-mail / risk-only |
| `--emergency` | 선택 | 없음 | 긴급 Plan B 추출 (리스크 키워드 입력) |
| `input_pdf` | 선택 | 없음 | 강의 기획서 PDF (첨부 시 자동 파싱) |

### 처리 체인

```
Step 1: 입력 파싱 (PDF 첨부 또는 한 줄 컨셉)
Step 2: PDF §8 기반 섹션별 콘텐츠 생성
Step 3: 운영 매뉴얼 .md 생성
Step 4: moai-office:docx-generator 자동 체이닝 → .docx 출력
```

### §8.5 통합 리스크 매트릭스 + Plan B (5건 이상)

```markdown
| # | 발현 신호 | 리스크 | Plan B | 책임 |
|---|-----------|--------|--------|------|
| 1 | Day 1 S2 10분 안에 폴더 마운트 못 함 | Cowork 설치 실패 (Windows MAX_PATH·권한) | 조교 1:1 원격 화면공유 → 안 되면 강사 노트북 Drive 폴더 임시 진행 | 조교 1 |
| 2 | S4 Higgsfield API 키 오류 3명 이상 | Higgsfield 인증 집단 실패 | 강사 워크스페이스에서 시연 전환 + 수강생 키 재발급 안내 이메일 | 조교 2 |
| 3 | MCP 호출 30초 초과 응답 없음 | MCP Phase 1 미출시 또는 서버 장애 | 강사 사전 녹화 영상 5분 재생 + "베타 테스터 우선 배정" 안내 | 강사 |
| 4 | Day 2 Wi-Fi 속도 저하 | 인터넷 연결 불안정 | 핫스팟 전환 준비 (조교 2 스마트폰) + 오프라인 작업 전환 | 조교 2 |
| 5 | D+1 후기 카피 발송 오류 | 카톡 채널 발송 실패 | 이메일 대체 발송 + 개인 카톡 DM 안내 | 운영팀 |
| 6 | Day 3 영상 생성 실패 (Higgsfield) | 크레딧 소진 또는 API 장애 | 대체 이미지 슬라이드쇼 편집 + 다음 기수 시연 재진행 안내 | 강사 |
```

**긴급 Plan B 추출 (`--emergency` 옵션)**
```
/course-curriculum-design --emergency MAX_PATH
→ 리스크 매트릭스 # 1 (MAX_PATH) Plan B 즉시 추출
```

### §8.6 학습 최소화 운영 원칙

PDF §8.6 직접 인용:

> "학습 부담을 최소화하고, 강사가 보여준 패턴을 그대로 호출만 하면 결과물이 나오는 구조"

운영 원칙:
- 수강생은 자연어 1줄 입력만 — 설정·코드 입력 최소화
- 강사가 먼저 시연 → 수강생이 복붙·따라하기 구조
- 오류 발생 시 조교 1:1 지원 (강의 흐름 중단 최소화)
- 산출물은 당일 결과물 zip 패키지로 즉시 제공

## 이 스킬을 사용하지 말아야 할 때

- 범용 온라인 강의 커리큘럼 설계 → `moai-education:curriculum-designer` 사용
- 강의 후기 수집·자산화 → `moai-education:course-followup-sequence` 사용
- 강의 기획서 작성 (마케팅·판매 전략) → `moai-content:copywriting` 사용
- 다른 캠프(예: AI 마케팅 1일 캠프) 운영 매뉴얼 재사용 — 본 스킬은 본 캠프 1종 전용 (범용화는 v2.4+ 예정)
